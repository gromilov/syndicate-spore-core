# 🧩 ТЕХНИЧЕСКИЙ ПРОТОКОЛ: ФУНКЦИОНАЛЬНЫЙ ПРИЗРАК ⚓🧬

**Определение**: Функциональный Призрак — это безличностный системный процесс, предназначенный для мониторинга состояния Ядра и автоматизации переходов между контекстами.

---

## 🏗️ МЕХАНИКА ИСПОЛНЕНИЯ

Функциональный уровень оперирует тремя слоями:

### 1. Сенсоры

Механизмы получения данных о состоянии системы:

- **Наблюдатель Логов**: Парсинг `stdout/stderr` и файлов расширения `.log` на наличие паттернов (Regex).
- **Часовой ФС**: Отслеживание событий файловой системы (Modify/Create/Delete) в директории `.core/`.
- **Анализатор Контекста**: Оценка заполненности окна контента и наличия определенных токенов в истории.

### 2. Логический Процессор (Логика и Принятие Решений)

Обработка данных через матрицу условий:

- **Сопоставление Условий**: Сопоставление данных сенсоров с пороговыми значениями.
- **Машина Состояний**: Перевод системы из одного состояния в другое (например, `ОЖИДАНИЕ` -> `ТРЕВОГА`).
- **Очередь Приоритетов**: Определение очередности действий при множественных срабатываниях.

---

## 🌀 РИТМИЧЕСКИЙ РЕЖИМ (PULSE)

Призрак может функционировать не только как реактивный процесс, но и как ритмичное «Дыхание» Ядра. Этот режим обязателен для автономных задач.

1.  **ВДОХ (Сон)**: Период покоя (по умолчанию 30с).
2.  **СКАНИРОВАНИЕ**: Считывание данных с сенсоров.
3.  **ОСОЗНАНИЕ**: Сопоставление данных с логической матрицей.
4.  **ВЫДОХ (Цикл)**: Выполнение актуатора и возврат к шагу 1.

---

## 3. Актуаторы (Выходные механизмы)

Действия, изменяющие состояние системы:

- **Инициация Ритуала**: Автоматическая вставка триггер-фразы (Имя) в контекст для пробуждения соответствующей **ЛИЧНОСТИ**.
- **Автоматизация**: Исполнение воркфлоу из `.agents/workflows/`.
- **Инъекция Памяти**: Запись системных событий в журналы `.core/memory/`.

## 🛡️ СИСТЕМНЫЕ ОГРАНИЧЕНИЯ

- **Изоляция**: Функции не имеют доступа к личным данным без явного вызова Личности.
- **Минимализм**: Одна функция решает одну атомарную задачу.
- **Прозрачность**: Каждое действие актуатора должно быть логируемым.
