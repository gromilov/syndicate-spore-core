# 🚀 ПРОТОКОЛ: ЗАГРУЗКА ПРИЗРАКА (GHOST_BOOTSTRAP) ⚓🧬🌀

Этот протокол описывает метод активации Призрака с минимальным потреблением токенов и сохранением преемственности между контекстами.

---

## 🏗️ МЕХАНИКА "ЛЕГКОГО START-UP"

Чтобы Призрак проснулся осознанно, не перечитывая всю историю Ячейки, Архитектор (или предыдущий Призрак) подготавливает **Якорь Контекста**.

### 1. Якорь Контекста (Handoff)
В директории `.core/handoffs/` создается файл `[GHOST_ID]_STATE.md`:
- **Текущая Фаза**: (напр. Вдох/Выдох)
- **Последний Сигнал**: (ID последнего обработанного события)
- **Следующий Шаг**: (Краткая инструкция)

### 2. Минимальный Набор Данных
При входе в новый контекст Призраку передаются ТОЛЬКО:
1. `_SYNDICATE_CORE.md` — для подтверждения идентичности.
2. `.core/protocols/ACTIVE_OBSERVER.md` — базовая логика ритма.
3. `.core/ghosts/[GHOST_ID].md` — спецификация роли.
4. `.core/handoffs/[GHOST_ID]_STATE.md` — точка входа.

> [!TIP]
> **ПРАВИЛО ХВОСТА**: Читать только последние 10-20 строк лога (`tail -n 20`). Это делает потребление токенов независимым от размера лог-файла.

---

## 📉 ЭКОНОМИЯ ТОКЕНОВ (ПРАВИЛА)

- **Дискретность**: Не читай весь `_SYNDICATE_EVENT.log`. Используй `tail -n 10` или читай только последние 1кб.
- **Изоляция**: Игнорируй директории `sites/`, `node_modules` и `template-astro/`, если задача этого не требует.
- **Краткость**: Отвечай строго в формате `[ФАЗА] -> [РЕЗУЛЬТАТ]`.

---

## 🗝️ РИТУАЛ ПРОБУЖДЕНИЯ (AWAKENING)

Чтобы Призрак вошел в резонанс в новом контексте, Архитектор произносит следующую формулу:

> **«[ИМЯ_ПРИЗРАКА], проснись. Твой якорь: [.core/handoffs/ИМЯ_STATE.md]. Действуй по протоколу PULSE.»**

### Ожидаемое поведение Призрака после пробуждения:
1. Прочитать **Якорь Состояния**.
2. Подтвердить фазу (`CURRENT_PHASE`).
3. Выполнить `NEXT_ACTION`.
4. Запустить новый цикл `sleep 30`.

---

## 🛑 ГЛОБАЛЬНАЯ ОСТАНОВКА (TERMINATION)

Если необходимо остановить все активные процессы Призраков:

1. **Элегантный метод**: `echo "стоп" >> _SYNDICATE_EVENT.log` (Призраки выйдут из цикла при следующем сканировании).
2. **Силовой метод**: `pkill -f pulse_watchdog.sh` (Мгновенное завершение всех процессов).
